## 📁 Примеры работ

### 1. Функциональное и UI/UX тестирование модуля "Комнаты" IoT-платформы Smart Home
Реальный кейс глубокого исследовательского тестирования веб-форм и логики элементов интерфейса в условиях отсутствия готовой документации.

* **Тест-дизайн:** Провела декомпозицию формы "Добавить комнату" и спроектировала матрицу из **82 высокодетализированных проверок** (покрыты UI/UX, кроссбраузерность и валидация).
* **Применение техник:** Использовала анализ граничных значений (BVA) для числовых полей (лимиты площадей от 1 до 999 999) и классы эквивалентности (EP) для верификации кодировок названий.
* **Результат:** Провела кроссбраузерный аудит (Chrome, Firefox, Яндекс.Браузер). Локализовала и оформила баг-репорты на критические дефекты логики (пропуск отрицательных значений площади) и несогласованность фронтенд-валидации.

### 2. Примеры SQL-запросов
В ходе учебных и практических задач регулярно применяю SQL для верификации данных на бэкенде.

```sql
-- Поиск пользователя в базе данных по его email
SELECT * 
FROM users 
WHERE email = 'test@example.com';

-- Вывод имён пользователей и дат их завершённых заказов (объединение таблиц)
SELECT 
    u.name,
    o.order_date
FROM users u
JOIN orders o ON u.id = o.user_id
WHERE o.status = 'completed';
```

### 3. Пример чек-листа (Авторизация)
Базовый пример высокоуровневой структуры проверок для модуля аутентификации:
* [ ] Проверить авторизацию с валидными учётными данными (Happy Path).
* [ ] Проверить вывод ошибки при вводе неверного пароля.
* [ ] Проверить логику восстановления доступа через форму "Забыли пароль".
